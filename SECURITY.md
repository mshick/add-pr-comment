# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 3.x     | :white_check_mark: |
| < 3.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please use one of the following methods:

1. **GitHub Security Advisories** (preferred): Use the [Report a vulnerability](https://github.com/mshick/add-pr-comment/security/advisories/new) button on the Security tab of this repository.
2. **Email**: Send details to the maintainer at <mshick@gmail.com>.

### What to include

- A description of the vulnerability
- Steps to reproduce the issue
- Any potential impact

### What to expect

- Acknowledgment within 48 hours
- A plan for a fix or mitigation within 7 days
- Credit in the release notes (unless you prefer to remain anonymous)
